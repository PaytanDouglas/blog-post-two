Setting up a virtual machine for a project.

This post is going to teach you how to create a project on a virtial machine. You should read this blog post if you are comfortable with the command line, have downloaded a program that can create virtual machines, and have a vagrant file ready to be installed on a virtual machine.

Before you begin: This blog post should only be read after you have downloaded VM software on your computer. This blog post is NOT made for people who are unfamiliar with Linux commands.

Vocab and basic commands you may need to know:
TermDefinition
virtual machineIn computing, a virtual machine is an emulation of a computer system.
They provide functionalities of physical computers
without actually being a physical computer.
cdChange Directory
vagrant upInstalls a vagrant environment onto the virtual machine.
vagrant sshSets up a secure shell inside the vagrant environment.
vagrant haltPauses the vagrant environment.
Should be used when taking a shorter-term break.
i.e. a food, coffee, or nap break.
vagrant suspendSaves progress on a project.
Should be used for longer-term breaks in working on the project.
vagrant destroyDestroys the vagrant environment.
The vagrant environment cannot be recreated.

Step 1: Getting into the vagrant folder.

From the command prompt you have to find where your vagrant file is. This is easiest if you have it on your desktop. You can 

cd Desktop
Then into the folder that holds your vagrant file if the file's not directly on your desktop.

Step 2: Installing vagrant.

vagrant up

This installs the specific vagrant box for the first time inside that folder. It may take a few minutes for your laptop to set up the vagrant box, so give it a few minutes.

Step 3: Securing the host.

vagrant ssh

This launches a secure shell host in the vagrant box and you should see your command line change from white to green, saying something similar to vagrant@vagrant-box-you-used:~$ with the ~ in blue and the $ in white.

It might take a minute, too, so be patient!

Step 4: Experimenting.

Now have some fun! The vagrant box is all set up and you can now work on projects with team members to your heart's content. Just remember that this environment is easily destroyed, so don't forget to be careful when coding!

Step 5: Stopping vagrant.

vagrant destroy / vagrant halt / vagrant suspend

If you're leaving the vagrant box for a lunch break or a quick stretch, use vagrant halt to press pause on your project.

If you're done with your project and aren't sure if you'll need it again, you can use vagrant suspend to save your progress. That way, if you come back to it later, all your data will be saved just how you left it.

ONLY USE vagrant destroy IF YOU ARE 1000% SURE YOU AREN'T GOING TO USE IT AGAIN!!! vagrant destroy does exactly what it sounds like -- it destroys the vagrant environment you created and all of your work. The only thing it keeps is the box you downloaded, so if you want to remove that too, use the command vagrant box remove.

Step 6: Repeating as necessary.

Experiment with vagrant before creating important files in the vagrant environment. I would recommend doing"test runs" until you get the hang of creating vagrant environments. They can be tricky, and even after doing one for the first time, you may still get confused when installing a different vagrant box.

With that all said, good luck and have fun!

Scafolding a project on a remote server using SSH and the command line interface.

This post is going to teach you how to create a project with multiple files on a remote server. You should read this blog post if you have a remote server you can access, files you want to start on that server, and are at least somewhat comfortable with SSH and the command line.

Before you begin: This blog post should only be read after you have downloaded terminal software on your computer. This blog post is made for people who are unfamiliar with Linux commands, but are familiar with html commands.

Vocab and basic commands you may need to know:
TermDefinition
scafoldingA framework for ; to create ; to add on to
mkdirMake Directory
cdChange Directory
touchCreates a file ; can be used to update timestamps
nanoCreates a file ; most useful when dealing with configuration files

Step 1: Logging into your terminal.

Log into your desired host (this is typically an IP address).

Once you enter your username and password for the terminal, the terminal should show 

your_username@your_server
and you can begin scaffolding.

Step 2: Creating a project folder and going to the folder.

In order to create a new directory, enter the line 

mkdir desired_directory_name
into your terminal.

This creates a new diretory (similar to a "folder" on your computer).
After the directory is made, you can enter that directory by typing 

cd desired_directory_name
into the terminal.

Step 3: Creating files (scripts) to put in your project folder.

This step assumes you aleady have created html files on a code editor. In order to create a new file for your project, use the command: 

touch desired_file_name

Once the file is created, you can copy nd paste your code in from your text editor.

Finally, you can run your code in step 4.

Step 4: Running those files.

Everything's led up to this, hasn't it?

To run the code from Step 3 from the file you created in Step 2 after logging into your terminal in Step 1 - we're finally here!

Type desired_file_name or ./desired_file_name into your terminal to see your code work it's magic!

And with that hopefully it turned out the way you wanted it to! Good luck coding!

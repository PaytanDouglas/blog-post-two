Scafolding a project on a remote server using SSH and the command line interface.

This post is going to teach you how to create a project with multiple files on a remote server. You should read this blog post if you have a remote server you can access, files you want to start on that server, and are at least somewhat comfortable with SSH and the command line.

Before you begin: This blog post should only be read after you have downloaded terminal software on your computer. This blog post is made for people who are unfamiliar with Linux commands, but are familiar with html commands.

Vocab and basic commands you may need to know:
TermDefinition
scafoldingA framework for ; to create ; to add on to
mkdirMake Directory
cdChange Directory
touchCreates a file ; can be used to update timestamps
nanoCreates a file ; most useful when dealing with configuration files

Step 1: Getting into the vagrant folder.

From the command prompt you have to find where your vagrant file is.

This is easiest if you have it on your desktop. You can cd Desktop then into the folder that holds your vagrant file if the file's not directly on your desktop.

Step 2: Installing vagrant.

vagrant up

This installs the specific vagrant box for the first time inside that folder. It may take a few minutes for your laptop to set up the vagrant box, so give it a few minutes.

Step 3: Securing the host.

vagrant ssh

This launches a secure shell host in the vagrant box and you should see your command line change from white to green, saying something similar to vagrant@vagrant-box-you-used:~$ with the ~ in blue and the $ in white.

It might take a minute, too, so be patient!

Step 4: Experimenting.

have some fun! The vagrant box is all set up and you can now work on projects with team members to your heart's content.

Step 5: Stopping vagrant.

vagrant destroy / vagrant halt / vagrant suspend

If you're leaving the vagrant box for a lunch break or a quick stretch, use vagrant halt to press pause on your project.

If you're done with your project and aren't sure if you'll need it again, you can use vagrant suspend to save your progress. That way, if you come back to it later, all your data will be saved just how you left it.

ONLY USE vagrant destroy IF YOU ARE 1000% SURE YOU AREN'T GOING TO USE IT AGAIN!!! vagrant destroy does exactly what it sounds like -- it destroys the vagrant environment you created and all of your work. The only thing it keeps is the box you downloaded, so if you want to remove that too, use the command vagrant box remove.

Step 6: Repeating as necessary.

Experiment with vagrant before creating important files in the vagrant environment.
